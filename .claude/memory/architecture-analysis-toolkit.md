# Analyzing this repo with LocalLLM_Code_Analysis

Verified: 2026-07-09 @ c0ef0e800 (toolkit @ its `koreader` preset, lua profile)

The LocalLLM_Code_Analysis toolkit (C:\Coding\rivaborn\LocalLLM_Code_Analysis)
can extract LSP-based architecture context for this repo's Lua frontend:

- Run from this repo's root:
  `powershell -File <toolkit>\llm_scripts\serena_extract.ps1 -Preset koreader -Workers 1..3`
  (needs lua-language-server; a portable copy lives in the toolkit's
  gitignored `.tools\`, or pass `-ServerPath`). Output lands in
  `architecture/.serena_context/` — gitignored here, resumable via its
  `.state/hashes.tsv`.
- The `koreader` preset scopes to `\.lua$` and excludes base/spec/platform/
  l10n/resources; profile = lua (lua-language-server reads this repo's
  `.luarc.json` automatically).
- Practical constraints on ROGSAG: each lua-language-server instance uses
  ~1-2.3 GB RAM and scans the whole workspace on startup (~30-60 s). With
  low free RAM prefer `-Workers 1` (~1 file/s once warm). Killed wrapper
  processes LEAK their language servers — check
  `Get-Process lua-language-server` and stop orphans if RAM vanishes.
- `.env` at this repo's root configures the LLM doc-gen stages against the
  homelab gateway (192.168.1.40; ollama, `qwen3.6:27b-64k`). Both `.env`
  and `architecture/` are gitignored.
- The committed event-dispatch xref (`doc/events_xref.md`, regenerated by
  `tools/event_xref.py`) is separate from and complements the toolkit
  output: LSP context gives per-file symbols/references; the xref maps the
  string-composed event dispatch the LSP cannot see.
